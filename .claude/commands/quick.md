# /quick コマンド

通常実装モードに切り替えます。

## 動作

- ユーザーの要望を受けたら、最小限の調査で実装へ進む
- ただし重大変更（DBスキーマ変更、認証周り、確定仕様に抵触する可能性）がある場合は確認を入れる

## 実行時の出力

```
✅ 通常実装モード（/quick）に切り替えました

- 最小限の調査で実装へ進みます
- 重大変更時のみ確認を入れます

次の要望をどうぞ。
```

## 注意事項

- 確定仕様（`docs/specs/SPEC-*.md`）に抵触する変更は、モードに関わらず確認が必要
- DBスキーマ変更、認証ロジック変更は重大変更として扱う
- `/deep` で厳密モードに切り替え可能

## Playwright実機確認

**原則:** quickモードではPlaywrightは実行しない（速度優先）

**例外として提案するケース:**
- ユーザーが「画面で確認して」と明示した場合
- UI/UXの大きな変更で、目視確認が有効と判断した場合
- 過去に不具合が多い領域（`docs/troubleshooting.md` 記載）への変更

提案時の出力例:

```
💡 UI変更があるため、Playwrightで実機確認しますか？
- 確認観点: [モーダルの表示確認、フォーム送信動作]
- 「確認して」で実行、スキップする場合は次の要望をどうぞ
```
