# 事業OS 開発ロードマップ — 優先順位と開発Phase

> **目的**: 754件の機能要求を開発Phaseに分類し、開発チームと優先順位を合意する
> **方針**: 営業・代理店領域 → 運用領域 → 拡張機能の順
> **作成日**: 2026-04-02
> **ステータス**: 開発チームとの合意待ち

---

## 開発方針

1. **OS基盤を最初に作る** — 全機能の土台（チャットUI・承認・通知・ダッシュボード・権限）がないと何も動かない
2. **営業→代理店→運用の順** — 売上を作る仕組みを先に、デリバリーを次に
3. **各Phaseの中でもMVP→拡張の段階を設ける** — 最初から全機能を作らない
4. **外部連携は各Phaseに組み込む** — 「連携セクション」を独立で作るのではなく、必要なタイミングで連携を実装

---

## 全体タイムライン

```
Phase 0 ─── OS基盤（全機能の土台）
  │
Phase 1 ─── 営業OS コア（リード→商談→提案→契約）
  │
Phase 2 ─── 営業分析・PDCA・経営ダッシュボード
  │
Phase 3 ─── 代理店・アライアンス管理
  │
Phase 4 ─── 運用OS コア（媒体管理・アラート・レポート）
  │
Phase 5 ─── 運用OS 拡張（ヘルススコア・クロスセル・CS・契約更新）
  │
Phase 6 ─── 経理・請求・支払い
  │
Phase 7 ─── クロスセル商材（リクブースト・AI面接等）
  │
Phase 8 ─── RPO特化・組織基盤・データ資産強化
```

## Phase間の依存関係

```
Phase 0（OS基盤）
  ├──→ Phase 1（営業コア）
  │      ├──→ Phase 2（営業分析・PDCA・経営ダッシュボード）
  │      │      └──→ Phase 3（代理店・アライアンス）
  │      │
  │      ├──→ Phase 4（運用コア）
  │      │      ├──→ Phase 5（運用拡張）
  │      │      └──→ Phase 6（経理・請求）
  │      │
  │      └──→ Phase 7（クロスセル商材）※Phase 4-5と並行可
  │
  └──→ Phase 8（RPO特化・組織基盤）※Phase 1-6安定後
```

**並行開発が可能な組み合わせ:**
- Phase 3 と Phase 4 は並行可能（営業チームと運用チームが別タスク）
- Phase 6 は Phase 4 完了後だが、Phase 2-3 とは並行可能
- Phase 7 は Phase 4-5 と並行で着手可能（リクブーストは独立性が高い）

## 開発チームとの合意ポイント

1. **Phase 0→1→2の順序は固定か？** 営業チームの業務が回ることが最優先
2. **Phase 2と4は並行開発するか？** チーム体制次第
3. **各Phase内の優先度（A/B/C）の粒度は適切か？**
4. **Phase 3の位置づけ** — 営業データが溜まる前に作っても意味がないが、早めに基盤だけ作っておくべきか
5. **クロスセル商材（Phase 7）の着手タイミング** — リクブーストは独立性が高いので早めに着手可能
