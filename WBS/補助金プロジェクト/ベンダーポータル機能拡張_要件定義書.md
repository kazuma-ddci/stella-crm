# ベンダーポータル機能拡張 — 包括要件定義書

## 1. 概要

### 1.1 背景
現在のベンダーポータル（`/hojo/vendor/[token]`）は2つのタブ（卸アカウント管理・助成金申請者管理）のみ。
Google スプレッドシート「コンサル案件管理シート」で管理していた以下の情報をCRMに統合する。

### 1.2 ゴール
- スプレッドシートの全タブをCRM内の個別ページとして再構築
- ベンダーポータルに新メニューバーを導入し、4つのセクションに整理
- スタッフ側管理画面にも対応する管理ページを追加
- ベンダー基本情報（ベンダー顧客リスト.xlsx）をCRMで管理

### 1.3 対象Excelシートとの対応

| Excelシート | CRM新セクション | 対象 |
|------------|----------------|------|
| ベンダー顧客リスト.xlsx | ベンダー基本情報（プロフィール） | スタッフ側 |
| 契約状況 | コンサル/BPO → 契約概要 | スタッフ+ベンダー |
| コンサル状況 | コンサル/BPO → コンサル活動記録 | スタッフ+ベンダー |
| 顧客情報_参考用/記入用（〜概要案内） | 助成金申請 → 概要案内フェーズ | スタッフ側 |
| 顧客情報_参考用/記入用（交付申請〜） | 助成金申請 → 交付申請フェーズ + 貸金セクション | スタッフ側 |

## 2. ベンダーポータル メニュー構造（新設）

### 2.1 現状
```
ベンダーポータル /hojo/vendor/[token]
├── タブ1: 卸アカウント管理
└── タブ2: 助成金申請者管理
```

### 2.2 新メニュー構造
タブ形式をやめ、**サイドバー or トップナビ**のメニューバーに変更。

```
ベンダーポータル /hojo/vendor/[token]
│
├── セキュリティクラウド卸
│   └── 卸アカウント管理（既存機能をそのまま移行）
│
├── 助成金申請
│   └── 助成金申請者管理（既存機能をベースに拡張）
│
├── コンサル/BPO
│   ├── 契約概要（契約状況シートの内容）
│   └── コンサル活動記録（コンサル状況シートの内容）
│
└── 貸金
    └── 貸付管理（貸付関連フィールド）
```

### 2.3 ベンダー vs スタッフの表示制御
- **ベンダー**: 自社データのみ閲覧・一部編集可
- **スタッフ**: 全ベンダーのデータを閲覧・編集可 + ベンダー非表示フィールドも管理

## 3. 機能A: ベンダー基本情報（スタッフ管理画面）

## 12. 検証方法

1. 各Phase完了時に `tsc --noEmit` で型エラーなし確認
2. マイグレーション適用 → Docker内generate → サーバー再起動
3. スタッフ側: 各管理ページでCRUD操作（作成・表示・編集・削除）の動作確認
4. ベンダーポータル: メニュー遷移 → 各セクション表示 → 権限制御（非表示フィールド確認）
5. ステータス管理: 選択肢の追加・編集・削除・並べ替え
6. コンサル活動記録: ベンダーが「課題/相談内容」「備考」のみ編集可能であること確認
