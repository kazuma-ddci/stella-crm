### 発見した問題点

**P1. 入金仕訳の借方/貸方が会計的に逆**
- `actions.ts:674-680`: 入金時（`isIncoming`）に元仕訳の**借方**を相手勘定として取得しているが、売上仕訳は通常「借方:売掛金 / 貸方:売上」。入金仕訳は「借方:普通預金 / 貸方:売掛金」であるべき。コードでは`isIncoming ? "debit" : "credit"`で借方の売掛金を取得し、それをcredit側に配置しているので**結果的には正しい**。ただしコメントが紛らわしい。→ minor

**P2. `findAccountByMethodType`が名前の部分一致で勘定科目を検索**
- `actions.ts:277-283`: `name: { contains: pattern }` でAccountを検索している。「未払金」で検索すると「仮受未払金」等の意図しない科目がヒットする可能性がある。本番運用ではAccountマスタのデータ次第で誤った科目が選ばれるリスクがある。→ minor（初期段階では問題になりにくい）

**P3. 自動生成仕訳の検索がdescriptionの文字列マッチに依存**
- `actions.ts:828-832`: 消込取り消し時の自動仕訳削除が`description: { contains: '消込#${id}' }`で検索している。descriptionはユーザー側で編集される可能性がある（仕訳管理画面から修正可能な設計）。もしdescriptionが変更されると、自動仕訳が削除されない。ただし`isAutoGenerated: true`と合わせてフィルタしているため、リスクは限定的。→ minor

**P4. 差額仕訳の`journalDate`が`new Date()`（現在日時）**
- `actions.ts:743`: 差額仕訳の日付が`new Date()`で現在のサーバー時刻。入出金日（`bankTransaction.transactionDate`）に合わせるべきか、仕訳日に合わせるべきかは運用次第だが、入金仕訳（L691）は`bankTransaction.transactionDate`を使っており不統一。→ minor

**P5. `tx as unknown as TxClient` のキャスト**
- `actions.ts:667, 766, 848`: Prismaのトランザクションクライアントを`unknown`経由でキャストしている。型安全性が損なわれるが、PrismaのトランザクションAPIの型制限上やむを得ない。→ minor
