## 発見された問題

### Major: `executeAutoJournal` に重複生成防止チェックがない

**ファイル**: `actions.ts:346-458`

`executeAutoJournal` は対象取引の `status: "awaiting_accounting"` のみをフィルタしているが、その取引に対して既に自動生成済みの JournalEntry (isAutoGenerated=true) が存在するかチェックしていない。

同じ取引IDで複数回呼び出すと、重複する下書き仕訳が生成される。取引ステータスは更新されないため、繰り返し実行可能な状態になっている。

**修正案**:
```typescript
// 既に自動生成仕訳がある取引を除外
const existingJournals = await prisma.journalEntry.findMany({
  where: {
    transactionId: { in: transactionIds },
    isAutoGenerated: true,
    deletedAt: null,
  },
  select: { transactionId: true },
});
const alreadyJournaled = new Set(
  existingJournals.map((j) => j.transactionId)
);

// フィルタリング
const eligibleTransactions = transactions.filter(
  (tx) => !alreadyJournaled.has(tx.id)
);
```

### Minor: 競合ダイアログで優先度変更が「推奨」止まり

**ファイル**: `auto-journal-table.tsx:955-1013`

設計書5.6に「保存は可能だが、priority設定を必須にする」とあるが、競合ダイアログは警告表示のみで、優先度の明示的な変更を促すUIがない（フォーム側のpriority欄で変更可能ではある）。実運用上は大きな問題にならない。

### Minor: `suggestRuleFromJournal` のUI未実装
