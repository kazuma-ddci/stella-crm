### 6. 検出した問題

#### **MAJOR: サマリー計算で taxType を考慮していない**

`page.tsx:64-67` と `transactions-table.tsx:155-162` の合計金額計算:

```typescript
// page.tsx:64-67
const totalRevenue = records
    .filter((r) => r.type === "revenue")
    .reduce((sum, r) => sum + r.amount + r.taxAmount, 0);
```

```typescript
// transactions-table.tsx:155-162
const revenue = filteredData
    .filter((r) => r.type === "revenue")
    .reduce((sum, r) => sum + r.amount + r.taxAmount, 0);
```

テーブルの各行の金額表示（`transactions-table.tsx:332-334`）では正しく `taxType` で分岐している:

```typescript
const totalAmount = row.taxType === "tax_excluded"
  ? row.amount + row.taxAmount  // 税抜+税額 = 税込
  : row.amount;                 // tax_included: amountが既に税込
```

しかし**サマリー計算ではこの分岐が抜けている**。`taxType === "tax_included"` の取引では `amount` にすでに税込金額が入っているため、`amount + taxAmount` は税額を二重計上する。
