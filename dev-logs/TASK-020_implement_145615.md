## 修正サマリー

### P1: 入金仕訳の自動生成 (Major)
**`actions.ts`** - `createReconciliation` 内で消込作成後に入金/支払仕訳を自動生成

- `findAccountByMethodType`: 決済手段の `methodType`（bank_account→普通預金、cash→現金、credit_card→未払金、crypto_wallet→仮想通貨）から対応する勘定科目を検索
- 入金（incoming）: `借方=決済手段の科目（普通預金等）/ 貸方=元仕訳の借方科目（売掛金等）`
- 出金（outgoing）: `借方=元仕訳の貸方科目（買掛金等）/ 貸方=決済手段の科目（普通預金等）`
- `isAutoGenerated: true`, `description` に `消込#${id}自動生成` を含めて取り消し時に特定可能に

### P2: Transaction.statusの更新 (Major)
**`actions.ts`** - `updateRelatedTransactionStatus` ヘルパー関数

- JournalEntryの `transactionId` / `invoiceGroupId` / `paymentGroupId` 経由で関連Transactionを特定
- 全関連仕訳の累計消込金額と取引金額を比較:
  - 消込合計 >= 取引金額 → `"paid"`
  - 0 < 消込合計 < 取引金額 → `"partially_paid"`
  - 消込合計 = 0 → `"journalized"`（取り消し時のリバート）
- `createReconciliation` と `cancelReconciliation` の両方で呼び出し

### P3: 月次クローズチェック (Major)
**`actions.ts`** - `checkMonthlyCloseForReconciliation` ヘルパー関数

- JournalEntry経由でTransaction→projectIdを取得
- 仕訳日と入出金日の両方の月がクローズ済みでないかチェック
- `accountingMonthlyClose` テーブル（既存パターン準拠）を参照
- `createReconciliation` と `cancelReconciliation` の両方で呼び出し

### Minor: 差額仕訳の論理削除
**`actions.ts`** - `cancelReconciliation` を改修

- 消込取り消し時に `description` が `消込#${id}` を含む `isAutoGenerated=true` の仕訳を `deletedAt` 設定で論理削除
- 差額仕訳の description を `消込#${id}差額` 形式に変更して追跡可能に

### Minor: 手動対応時の案内
**`reconciliation-modal.tsx`** - `handleSubmit` を改修

- `differenceType === "manual"` 時にサーバーに `differenceType: "manual"` を送信
- 成功後に「差額の仕訳を手動で作成してください」のトースト + 仕訳画面へのリンクを表示
