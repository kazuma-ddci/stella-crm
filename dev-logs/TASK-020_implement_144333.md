TASK-020: 消込処理画面の実装が完了しました。以下が実装内容のサマリーです。

## 実装ファイル一覧

### 1. `src/app/accounting/reconciliation/actions.ts` - Server Actions
- **`getUnmatchedBankTransactions()`** - 未消込/部分消込の入出金を取得（既消込金額を計算して残額ありのもののみ返す）
- **`getUnmatchedJournalEntries()`** - 未消込/部分消込の確定済み仕訳を取得（借方合計と既消込金額を比較）
- **`getReconciliations()`** - 消込履歴を取得（実行者、仕訳、入出金の詳細を含む）
- **`getReconciliationFormData()`** - 勘定科目マスタデータ（差額仕訳用）
- **`createReconciliation()`** - 消込作成（仕訳残額・入出金残額のバリデーション付き。差額処理として振込手数料/値引きの差額仕訳自動生成にも対応）
- **`cancelReconciliation()`** - 消込取り消し（物理削除 → 入出金と仕訳が未消込に戻る）

### 2. `src/app/accounting/reconciliation/page.tsx` - サーバーコンポーネント
- `Promise.all()` で4種のデータを並列取得し、クライアントコンポーネントに渡す

### 3. `src/app/accounting/reconciliation/reconciliation-table.tsx` - メインUI
- **消込操作エリア**: 選択した入出金・仕訳をBadgeで表示、消込実行ボタン
- **未消込入出金テーブル**: ラジオボタンで選択、日付/区分/金額/未消込額/取引先/決済手段を表示
- **未消込仕訳テーブル**: ラジオボタンで選択、日付/摘要/金額/未消込額/科目を表示
- **消込履歴テーブル**: 消込日時/入出金/仕訳/消込金額/実行者/取り消しボタン
- 各テーブルに検索フィルタ付き

### 4. `src/app/accounting/reconciliation/reconciliation-modal.tsx` - 消込実行モーダル
- 選択された入出金・仕訳の概要表示
- **金額不一致検知**: 入出金残額と仕訳残額が異なる場合に警告を表示
- **差額処理**: 4種類の処理方法を選択可能
  - **一部入金**: 消込額を入金額で登録、残額は未消込
  - **振込手数料**: 差額仕訳（支払手数料/売掛金）をプレビュー表示＋科目・金額の修正可能
  - **値引き**: 差額仕訳（売上値引/売掛金）をプレビュー表示＋科目・金額の修正可能
  - **手動**: 後で差額仕訳を別途作成
- **ルール追加提案**: 振込手数料/値引きで消込完了時に「ルールに追加しますか？」ダイアログを表示
