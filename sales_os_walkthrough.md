# 営業OS ウォークスルー — 1案件の流れを体験する

> **目的:** 営業OSが実現した場合、商談予約から契約まで「営業が何をして、OSが何をするか」を1つの案件で追体験する
> **例の企業:** 株式会社山田建設（建築系・施工管理の採用に困っている中小企業）
> **関連:** 詳細設計は `sales_os_ideal_flow.md` を参照

---

## ストーリーの登場人物

- **齋藤さん（営業）** — ステラの営業担当
- **山田社長（お客様）** — 株式会社山田建設の代表。施工管理の採用に困っている
- **OS** — ステラの事業OS

### 10:00〜10:40 — 商談の進行

齋藤さんはOSの診断画面を手元のPCで開いている（画面共有はまだしない）。

**商談の会話（イメージ）:**

> **齋藤:** 山田社長、本日はお時間いただきありがとうございます。施工管理の採用でお困りとのことですが、今どんな状況ですか？

> **山田社長:** うちは今48人くらいの会社でね。施工管理を年間で5人くらい採りたいんだけど、全然応募が来なくて。Indeedに出してるんだけど、1ヶ月で2〜3件しか応募がない。

齋藤さんがOSの診断画面に入力していく:

```
┌─── ヒアリング入力 ──────────┐  ┌─── 診断結果（リアルタイム）──┐
│                            │  │                            │
│ 業種: [建築系 ▼]（済）     │  │ ← 入力するたびに           │
│ 職種: [施工管理 ▼]（済）   │  │    ここの数字が変わる       │
│ 地域: [関東 ▼]（済）       │  │                            │
│ 年間採用目標: [5] 人  ★今入力│  │ CPA試算                    │
│ 採用課題: ☑応募が来ない ★今入力│ │  min: ¥5,000               │
│ 現在のチャネル: ☑Indeed  ★今入力│ │  mid: ¥10,000              │
│ 月間予算: [__] 万円        │  │  max: ¥15,000              │
│ 現在の採用単価: [__] 万円  │  │                            │
│                            │  │ 推奨プラン: —（予算入力待ち）│
│                            │  │                            │
└────────────────────────────┘  └────────────────────────────┘
```

> **齋藤:** 年間5名ですね。ちなみに紹介会社さんは使われていますか？

> **山田社長:** 前に使ったことあるよ。1人採用するのに120万かかって、それっきりやめた。

齋藤さんが入力:
```
  現在の採用単価: [120] 万円（紹介会社利用時）
  現在のチャネル: ☑Indeed ☑紹介会社（過去）
```

診断結果がリアルタイムで更新される:
```
  年間コスト比較:
    業界平均（紹介）: 120万 × 5名 = 600万/年
    採用ブースト月額: 15万 × 12ヶ月 + 広告費 = 約220万/年
    → 年間 約380万円の削減見込み  ← この数字がリアルタイムで出る
```

> **齋藤:** 山田社長、ちょっとこの画面見ていただけますか？

齋藤さんがMeetの画面共有をONにして、診断結果の右側をお客様に見せる。

> **齋藤:** 今お伺いした条件だと、業界平均で年間600万かかっているところを、約220万に抑えられる試算が出ています。年間380万の削減です。

> **山田社長:** 380万！？ そんなに変わるの？

> **齋藤:** はい。Indeedの運用を最適化すると、応募単価がこれくらい（CPA ¥5,000〜¥15,000）まで下がる実績があります。建築系の施工管理で、実際にこれくらいのコストで採用できているお客様がいらっしゃいます。

> **山田社長:** ……これ、もうちょっと詳しい資料もらえる？

> **齋藤:** もちろんです。この診断結果をそのまま提案書にしてお送りしますね。

> **山田社長:** うん、社内で検討したいから、上に見せる用の資料もほしいな。

> **齋藤:** 稟議書もお作りしますね。
