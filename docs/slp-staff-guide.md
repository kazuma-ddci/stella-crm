## 1. 全体の流れ

お客様が公式LINEを通じて以下のステップを進めます。

```
① 公式LINE友達追加
   ↓
② リッチメニューの「組合入会はこちら」から入会フォーム送信
   ↓
③ 電子契約書（CloudSign）がメールで届く → 締結
   ↓
  ※ 締結後、リッチメニューが開放される
  ※ 概要案内予約・お友達紹介URL等が使えるようになる
   ↓
④ 概要案内の予約（事業者情報の登録）
   ↓
⑤ 概要案内の実施 → 完了マーク
   ↓
⑥ 導入希望商談の予約
   ↓
⑦ 導入希望商談の実施 → 完了マーク
   ↓
⑧ 書類提出
```

**重要:** 組合員入会（②③）が完了するまで、概要案内の予約やお友達紹介などの機能は使えません。リッチメニューは契約締結後に開放されます。

各ステップはプロライン（LINEの自動化ツール）とCRM（顧客管理システム）が自動連携して進行します。

## 2. お客様がやること

### 2-1. 公式LINE友達追加

- お客様が公式LINEを友達追加すると、CRMのLINE友達一覧に自動登録されます
- 紹介者がいる場合、紹介者にLINE通知が届きます
- この時点ではリッチメニューに「組合入会はこちら」のみ表示されます
- 概要案内予約やお友達紹介URLなどの機能はまだ使えません

### 2-2. 組合員入会フォーム（最初にやること）

お客様がリッチメニューの「組合入会はこちら」をタップして入会フォームを開きます。

**送信後の流れ:**
1. 「送信確認中です...」と表示（約10秒間）
2. メールが正常に送付された場合 → 「契約書を送付しました」
3. メールが届かなかった場合 → 「メールアドレスが間違っている可能性があります」
   - お客様は「変更する」でメアドを修正して再送付できます
   - 「間違いない」を選択した場合はスタッフ対応になります

**契約書の締結:**
- お客様がメールで届いた契約書に電子署名すると、CRMの組合員名簿のステータスが「組合員契約書締結」に自動更新されます
- **締結完了後、リッチメニューが開放され**、概要案内予約・お友達紹介URL・動画閲覧などが使えるようになります

**再訪問時:**
- 前回入力したデータがフォームに自動入力されています
- メール不達の場合はメアド確認画面が優先表示されます
- 契約締結済みの場合は「すでに契約済みです」と表示されます

### 2-3. 概要案内の予約（契約締結後に利用可能）

お客様がリッチメニューから「概要案内予約」を開くと、CRMの予約ページが表示されます。

**予約ページでの操作:**
1. **法人か個人事業主かを選択**
2. **事業者名を入力**（法人なら企業名、個人事業主なら屋号や個人名）
3. プロラインの予約カレンダーが開き、日時を選択して予約確定

**予約確定後に自動で起きること:**
- CRMの事業者名簿に新しいレコードが作成される
- ステータスが「予約中」になる
- 紹介者にLINE通知が届く
- プロライン担当者設定にマッピングがあれば、担当者が自動で入る

**既にCRMに登録されている事業者の場合:**
- 「あなたが担当している事業者」リストから選択して予約できます

### 2-4. 概要案内の予約変更・キャンセル

- **変更:** プロラインの予約履歴から日時を変更 → CRMの予約日時が自動更新される
- **キャンセル:** プロラインの予約履歴からキャンセル → CRMのステータスが「キャンセル」に変わる
- キャンセル後は再度予約が可能です

### 2-5. 導入希望商談の予約

- 概要案内が「完了」になっている事業者のみ予約可能です
- 概要案内未完了の事業者は「まだ概要案内が完了していない事業者」として表示され、予約できません
- 予約・変更・キャンセルの仕組みは概要案内と同じです

### 2-6. 書類提出

お客様が公式LINEから書類提出フォームを開き、各書類をアップロードします。
事業者ごとに書類が管理されます。

## 3. CRM管理画面の機能

### 3-1. 事業者名簿

**場所:** サイドバー → 公的(SLP) → 事業者名簿

事業者の一覧が表示されます。

**一覧画面の機能:**
- **バッジ表示:** 事業者名の横に「法人」「個人」のバッジが表示されます
- **フィルタ:** 事業形態（全て/法人/個人事業主/未設定）でフィルタリング可能
- **検索:** 事業者名、事業者No.、担当営業で検索可能

**詳細画面の機能:**
- **事業形態の設定:** 法人/個人事業主のラジオボタンで切り替え
  - 法人 → 企業名、法人番号、企業電話、企業メール、本店所在地
  - 個人事業主 → 屋号(個人名可)、事業用電話、事業用メール、事業用住所
- **代表者の設定:** 担当者リストから代表者を選択すると、名前・電話・メールが自動入力されます。代表者情報と担当者情報は双方向で同期されます。
- **ステータス管理:** 概要案内・導入希望商談のステータスを変更できます（変更理由の入力が必須）
- **完了処理:** 概要案内/商談が完了したら、完了ボタンからお客様にお礼メッセージを送信できます

### 3-2. 組合員名簿

**場所:** サイドバー → 公的(SLP) → 組合員名簿

組合員入会フォームから登録されたお客様の一覧です。
契約書の送付状態（未送付/送付済/締結済/送付エラー）が確認できます。

### 3-3. 代理店管理

**場所:** サイドバー → 公的(SLP) → 代理店管理

代理店の情報を管理します。

- **法人/個人事業主の切り替え:** Switchで切り替えると、表示されるフィールドが変わります
- **契約ステータス:** スタッフが自由に追加・編集・削除・並び替えできます（マスタ管理）
- **担当者管理:** 代理店ごとに担当者を登録できます
- **親子階層:** 代理店の親子関係をツリー構造で管理できます

### 3-4. LINE友達一覧

**場所:** サイドバー → 公的(SLP) → LINE友達

公式LINEの友達一覧です。プロラインから自動同期されます。

### 3-5. プロライン担当者設定

**場所:** サイドバー → 公的(SLP) → 設定 → プロライン担当者

プロラインの予約フォームで選択される担当者名と、CRMのスタッフIDを紐付ける設定です。

**重要:** この設定がないと、予約時に担当者が自動入力されません。事業者詳細画面に「プロラインからの担当者名: ○○（マッピング未登録）」と表示された場合、ここでマッピングを追加してください。

### 3-6. 重複検出

事業者名簿で重複の可能性があるレコードを自動検出します。

**検出条件:**
- **法人番号が一致**（最優先・確実）
- **事業者名の類似**（3文字以上共通）
- **電話番号の一致**
- **住所の類似**
- **個人事業主の代表者名+電話番号が一致**

重複候補が見つかると、事業者詳細画面に警告が表示されます。必要に応じて事業者の統合（マージ）を行ってください。

## 4. プロラインとCRMの自動連携

### お客様の操作 → CRMへの自動反映

| お客様の操作 | CRMに起きること |
|---|---|
| 公式LINEを友達追加 | LINE友達一覧に追加 |
| 概要案内を予約 | 事業者名簿にレコード作成、ステータス「予約中」 |
| 概要案内を日時変更 | 予約日時が更新 |
| 概要案内をキャンセル | ステータス「キャンセル」、予約情報クリア |
| 導入希望商談を予約 | 商談ステータス「予約中」 |
| 導入希望商談を日時変更 | 商談日時が更新 |
| 導入希望商談をキャンセル | 商談ステータス「キャンセル」、商談情報クリア |
| 入会フォームを送信 | 組合員名簿にレコード追加 |

### CRMの操作 → お客様へのLINE通知

| CRMの操作 | お客様/紹介者に届くもの |
|---|---|
| 概要案内を「完了」にする | 受講者にお礼メッセージ(LINE)、紹介者に完了通知(LINE) |
| 商談を「完了」にする | 受講者にお礼メッセージ(LINE) |
| 入会フォーム送信（自動） | お客様に電子契約書（メール） |
| 契約書メール不達（自動） | お客様にLINE通知「メールが届きませんでした」 |

## 5. 電子契約書（CloudSign）の仕組み

### 送付の流れ

1. お客様が入会フォームを送信
2. CRMがCloudSignのAPI経由で契約書を自動送付
3. お客様のメアドに契約書が届く
4. お客様が契約書に電子署名
5. CRMの組合員名簿のステータスが「組合員契約書締結」に自動更新

### メール不達時の流れ

1. CloudSignが「メールが届かなかった」とCRMに通知
2. CRMが組合員名簿の「不達フラグ」をONにする
3. **入会フォーム送信直後（10秒以内）に不達検知:**
   - お客様に「メアドが間違っている可能性があります」画面を表示
   - お客様がメアドを修正して再送付可能
4. **10秒以降に不達検知:**
   - お客様にLINE通知で「メールが届きませんでした」と通知
   - お客様が再度フォームを開くとメアド確認画面が表示される

### メアド変更の回数制限

- **メール不達によるメアド修正:** 回数制限なし
- **正常送付後のメアド変更:** 2回まで（それ以上は公式LINEへお問い合わせ）

### リマインド

- **お客様自身:** 入会フォームの「リマインド送付を希望する」ボタンから送信
- **自動:** 毎日朝10時に、送付から一定日数経過した未締結者に自動リマインド

## 7. 自動化エラーの確認と対応

**場所:** サイドバー → 公的(SLP) → 自動化エラー

プロラインやCloudSignとの連携で問題が発生した場合、ここに記録されます。

### よく表示されるエラーと対応

| エラーメッセージ | 意味 | 対応方法 |
|---|---|---|
| **マッピング未登録のプロライン担当者名を受信** | プロラインの担当者名がCRMのスタッフと紐付いていない | 「設定 → プロライン担当者」からマッピングを追加 |
| **紹介者が組合員名簿に未登録のため、紹介者なしで登録しました** | 紹介者がまだ入会フォームを提出していない | 紹介者が登録完了後、組合員名簿の「紹介者」を手動で設定 |
| **契約書送付失敗** | CloudSignのAPI設定エラーまたは一時的な障害 | CloudSignの設定を確認し、必要に応じて手動で再送付 |
| **契約書再送付失敗（メール不達修正）** | メアド修正後の再送付も失敗 | お客様に公式LINEで連絡し、正しいメアドを確認 |
| **CloudSignメール送信失敗** | メアドのタイプミスやドメインの問題 | 組合員名簿でメアドを修正し、手動で契約書を再送付 |
| **組合員への契約書リマインド送信に失敗** | CloudSignのリマインド送信エラー | CloudSignの契約書ステータスを確認 |
| **プロラインフォーム送信失敗** | プロラインへの通知送信が失敗 | 自動リトライはないため、必要に応じて手動でLINEメッセージ送信 |

## 8. よくあるトラブルと対処法

### Q. 予約したのにCRMに反映されない

**原因:** プロラインのwebhookが正常に届いていない可能性があります。

**対処:**
1. 自動化エラーページを確認
2. エラーがなければ、プロラインの予約履歴を確認し、予約が正常に完了しているか確認
3. 必要に応じてCRM上で手動でステータスと日時を設定

### Q. キャンセルしたのに予約画面で「予約中」のまま

**原因:** キャンセルwebhookが正しく処理されなかった可能性があります。

**対処:** CRM上でステータスを手動で「キャンセル」に変更（変更理由を入力）

### Q. 担当者が自動入力されない

**原因:** プロラインの担当者名がCRMのスタッフに紐付いていません。

**対処:**
1. 事業者詳細画面の担当者欄に「プロラインからの担当者名: ○○（マッピング未登録）」が表示されている場合
2. 「設定 → プロライン担当者」から担当者名のマッピングを追加
3. 次回以降の予約から自動入力されるようになります

### Q. 入会フォームを送信したのに組合員名簿にレコードがない

**原因:** 紹介者が未登録などの理由で登録処理がエラーになった可能性があります。

**対処:**
1. 自動化エラーページを確認（日本語のエラーメッセージが記録されています）
2. エラー内容に応じて手動で対応

### Q. 契約書メールが届かない

**原因:** メールアドレスのタイプミス、迷惑メールフィルタ、受信サーバーの問題など。

**対処:**
1. 組合員名簿で「不達フラグ」がONになっていないか確認
2. ONの場合、正しいメアドに手動で契約書を再送付
3. お客様にも迷惑メールフォルダの確認を依頼

### Q. 概要案内が完了したのに導入希望商談の予約画面に表示されない

**原因:** CRM上で概要案内ステータスが「完了」になっていない可能性があります。

**対処:** 事業者詳細画面で概要案内ステータスが「完了」になっているか確認し、なっていなければ完了処理を実行

### Q. 同じ事業者が2つ登録されてしまった

**原因:** お客様が2回予約するなどして重複登録された可能性があります。

**対処:**
1. 事業者詳細画面に「この事業者には○件の重複候補があります」と表示される
2. 重複候補を確認し、必要に応じて事業者の統合（マージ）を実行

### Q. 法人/個人事業主の区分が未設定のレコードがある

**原因:** 法人/個人事業主対応以前に登録されたレコードです。

**対処:** 事業者詳細画面で事業形態（法人/個人事業主）を手動で設定してください。

## 9. 資料（PDF）の管理

### アップロード方法

**場所:** 固有設定 → 資料管理

1. 「資料をアップロード」ボタンをクリック
2. PDFファイルを選択（最大50MB）
3. アップロードすると自動的に「アクティブな資料」に切り替わります

**ポイント:**
- アクティブな資料は1つだけです。新しい資料をアップロードすると、前の資料は非アクティブになります
- お客様には公式LINEからリッチメニューを通じてアクティブな資料のURLが配信されます
- 非アクティブな資料もOS上で履歴として残ります

### お客様への配信

お客様がLINEのリンクを開くと、認証（UID + LINE名）を経て資料が表示されます。
- お客様ごとにウォーターマーク（透かし）が入ります
- スクリーンショットや右クリック保存の抑制機能があります

---

## 10. 動画の管理

### アップロード方法

**場所:** 固有設定 → 資料管理（動画タブ）

1. 「動画をアップロード」ボタンをクリック
2. 動画ファイルを選択（MP4/WebM/MOV、最大1GB）
3. アップロードすると自動的に「アクティブな動画」に切り替わります

**ポイント:**
- 資料と同じく、アクティブな動画は1つだけです
- お客様にはLINEのリンクを通じて配信されます

### お客様への配信

お客様がLINEのリンクを開くと、ウォーターマーク付きのビデオプレーヤーで再生されます。
- お客様のLINE名とIDが透かしとして表示されます（流出防止）
- ダウンロードボタンは非表示、右クリックも無効です
- 動画のシーク（早送り・巻き戻し）は可能です

## 11. 閲覧ログ（アクセスログ）

### 確認方法

**場所:** 固有設定 → 資料管理 → 「アクセスログ」タブ

資料（PDF）と動画を閲覧したお客様のログが記録されています。

**記録される情報:**
- お客様のLINE友達ID・LINE名
- 閲覧した種別（資料 or 動画）
- アクセス日時
- IPアドレス・ブラウザ情報

**活用方法:**
- お客様が資料・動画を閲覧したかの確認
- 万が一の流出時、ウォーターマーク+IPアドレスで流出元を特定
