# ダッシュボードの自社開発 vs SaaS：なぜ自社開発の方が経営に効くのか

## はじめに

「ダッシュボードをユーザーが自由にカスタマイズできるようにしたい」という要望に対して、SaaS製品のダッシュボード機能と自社開発（stella-crm）の違いを整理し、**なぜ自社開発のダッシュボードの方が経営・事業に効果を発揮できるのか**を説明します。

---

## 1. SaaSのダッシュボードの実態と弱み

### SaaSダッシュボードの仕組み

Salesforce、HubSpot、Monday.com等のSaaS製品には、ダッシュボード機能が搭載されています。ユーザーは用意されたウィジェット（グラフやカード）を選び、ドラッグ&ドロップで配置できます。

一見すると自由度が高く見えますが、**表示できるのはSaaS側が用意した汎用的な指標だけ**です。

### SaaSで表示できるもの（汎用指標）

- 売上の合計・推移
- 顧客数・リード数
- 案件のステータス別件数
- 担当者別の活動量
- 一般的なファネル（リード→商談→成約）

### SaaSでは表示できないもの（御社独自の経営指標）

| 指標 | なぜSaaSでは出せないか |
|------|----------------------|
| **代理店ごとのROI**（紹介料に対する売上回収率） | 代理店との契約形態・手数料計算が御社独自 |
| **アライアンス部門のSQL化率** | 「SQL」の定義自体が会社ごとに異なる |
| **運用開始リードタイム**（契約→運用開始までの日数） | 御社の業務フロー固有のステータス遷移に基づく |
| **粗利の着地予測**（固定費・変動費を考慮した予測値） | 御社独自の費用構造と計算ロジックが必要 |
| **未入金アラート**（請求書の支払期限と入金状況の突合） | 経理業務のフローと判定条件が御社固有 |
| **リードソース別の費用対効果予測** | チャネル定義・費用按分ルールが御社独自 |

**SaaSのダッシュボードは「どの会社にも当てはまる最大公約数」しか出せません。**

経営判断に本当に必要なのは「うちの会社特有の数字」であり、それは汎用ツールでは原理的に出せないのです。

### SaaSダッシュボードのよくある末路

1. 導入時：「ダッシュボード機能があるから便利そう」と期待する
2. 設定時：「見たい数字がウィジェットにない」と気づく
3. 運用時：「結局Excelで集計している」「ダッシュボードは誰も見ていない」

これは導入企業の怠慢ではなく、**SaaSの構造的な限界**です。

## 2. 自社開発（stella-crm）ダッシュボードの強み

### 御社の業務ロジックがそのまま数字になる

stella-crmのダッシュボードでは、以下のような**御社固有の経営指標**がリアルタイムで表示されています。

- **KGI 5指標**：売上・粗利・新規契約数を目標値と比較し、達成率をリアルタイム表示
- **部門別KPI**：アライアンス・セールス・バックオフィス・CSの各部門に最適化された指標
- **セールスファネル**：御社の営業プロセスに合わせたステージ定義と転換率
- **代理店ROI**：紹介料と売上を突合した、代理店ごとの投資対効果
- **売上推移＋粗利着地予測**：計画vs実績に加え、固定費を考慮した着地予測
- **リードソース別予実**：チャネルごとの獲得コストと成果を可視化

**これらはすべて、御社の業務データと計算ロジックに基づいて算出されています。** SaaSのウィジェットライブラリには存在しない指標です。

### 経営・事業への効果

| 観点 | SaaSダッシュボード | stella-crmダッシュボード |
|------|-------------------|------------------------|
| **意思決定の速度** | 汎用指標を見た後、Excelで詳細分析が必要 | 画面を開いた瞬間に判断できる数字が並ぶ |
| **問題の早期発見** | 「売上が下がった」はわかるが原因がわからない | 「どの代理店経由が落ちているか」「どのステージで詰まっているか」まで一目でわかる |
| **部門間の連携** | 各部門が別々のレポートを見ている | 全部門のKPIが一画面に集約され、部門横断で状況が把握できる |
| **アラートの精度** | 「売上が目標以下」のような大雑把な通知 | 「未入金が○日超過」「特定代理店のROIが悪化」等の具体的な検知 |
| **データの信頼性** | SaaSにデータを正しく入力する運用が前提 | 業務の中で自然に蓄積されたデータをそのまま表示 |

## 3. 「ユーザーが自由にダッシュボードを作る」ことが難しい理由

### スタッフの皆さんへ：なぜ「自由に組み替え」が難しいのか

ダッシュボードに表示されている数字は、**画面に見えている部分は氷山の一角**です。

例えば「代理店ROI」というカードに「156%」と表示されているとき、裏側では以下のような処理が動いています。

```
【画面に見えるもの】
┌─────────────────┐
│  代理店ROI       │
│  156%    ▲12%   │
│  ██████████░░░   │
└─────────────────┘

【裏側で動いているロジック（見えない部分）】

① 対象月の代理店契約を全件取得
② 各契約の紹介手数料を集計（税込→税抜変換含む）
③ 同じ代理店経由の売上を全件取得
④ 売上から原価を差し引いて粗利を算出
⑤ 粗利 ÷ 紹介手数料 でROIを計算
⑥ 前月のROIも同様に計算して比較
⑦ 結果をカードのフォーマットに整形して表示
```

この①〜⑦の処理は、**プログラムのコード（約50〜100行）として書かれています。**

「代理店ROI」一つとっても、以下のような御社固有の判断が埋め込まれています。
- 「紹介手数料」はどのテーブルのどの項目を見るか
- 税込金額をどう扱うか
- 「売上」の定義（確定分だけか、見込みも含むか）
- 前月比較の基準日をどうするか

### なぜドラッグ&ドロップでは解決しないか

ドラッグ&ドロップで変えられるのは**カードの「位置」と「大きさ」だけ**です。

```
【ドラッグ&ドロップで変えられること】
┌──────┐┌──────┐        ┌──────┐┌──────┐
│ 売上  ││ 粗利  │   →    │ 粗利  ││ 売上  │  ← 並び替え
└──────┘└──────┘        └──────┘└──────┘

【ドラッグ&ドロップでは変えられないこと】
「売上」の数字を「先月比」から「前年比」に変えたい
「代理店別」ではなく「業種別」で見たい
「粗利率が30%以下の案件だけ」をリスト表示したい
   → これらはすべて裏側のロジック（コード）の変更が必要
```

### SaaSが「自由にカスタマイズできます」と言える理由

SaaSのウィジェットは**汎用的＝浅い**からこそ、設定画面で切り替えられます。

```
【SaaSのウィジェット例：売上グラフ】
データソース：売上テーブル → 固定（変更不可）
集計方法：  合計 / 平均 / 件数 → 選択式
期間：      日 / 週 / 月 / 四半期 → 選択式
グラフ種類：棒 / 線 / 円 → 選択式
```

やっていることは「同じデータの見せ方を変える」だけです。

一方、stella-crmのウィジェットは**業務ロジックが組み込まれている＝深い**ため、設定画面だけでは切り替えられません。

```
【stella-crmのウィジェット例：代理店ROI】
データソース：契約テーブル + 請求テーブル + 代理店テーブル → 3テーブル結合
計算ロジック：粗利算出 → ROI算出 → 前月比較 → 独自の計算式
表示条件：  アクティブな代理店のみ → 業務ルールに基づくフィルター
アラート：  ROI100%未満で警告 → 御社の基準値
```

**「深い」からこそ経営に効く数字が出せるが、「深い」からこそ設定画面だけでは変えられない。** これがトレードオフです。

## 4. stella-crmのダッシュボードを今後どう進化させるか

### 推奨アプローチ：「表示の自由度」と「データの深さ」を分けて考える

#### すぐできること（表示の自由度を上げる）

| 機能 | 内容 |
|------|------|
| **ウィジェットの表示/非表示** | 自分に不要なカードを隠せる |
| **レイアウトの変更** | ドラッグ&ドロップでカードの配置を変えられる |
| **ビューの保存** | 「代表用」「営業用」「CS用」等のプリセットを用意 |
| **フィルターの変更** | 表示期間や対象の切り替え |

これらは裏側のロジックに触れないので、ユーザーが自由に操作できます。

#### 要望ベースで追加すること（データの深さを拡張する）

| 要望の例 | 対応 |
|---------|------|
| 「業種別の売上も見たい」 | 新しいウィジェットを開発して追加 |
| 「契約更新率をダッシュボードに出したい」 | 計算ロジックを開発してウィジェット追加 |
| 「未入金アラートの閾値を変えたい」 | 設定画面で閾値だけ変更可能にする |

**新しい「データの切り口」が欲しい場合は開発が必要ですが、stella-crmではウィジェットの追加を効率化する仕組み（テンプレート化）を構築することで、1つあたりの開発時間を大幅に短縮できます。**

### SaaSにはできない、stella-crmだからできること

- 経営会議で「この数字が見たい」と言われたら、**翌日には**ダッシュボードに反映できる
- 業務フローが変わったら、**計算ロジックごと更新**できる
- 御社の経営指標の定義が変わっても、**コードを修正するだけ**で全画面に即反映
- SaaSでは絶対に出せない**御社固有のクリティカルな指標**を表示できる

---

## まとめ

| 観点 | SaaS | stella-crm |
|------|------|------------|
| ダッシュボードの自由度 | 高い（並べ替え・表示切替が簡単） | 現状は固定（今後改善可能） |
| 表示できる指標の深さ | **浅い**（汎用指標のみ） | **深い**（御社固有の経営指標） |
| 経営判断への貢献度 | **低い**（結局Excelが必要になりがち） | **高い**（画面を見れば判断できる） |
| 新しい指標の追加 | 不可能（SaaS側に機能がなければ出せない） | **可能**（開発で対応） |
| ユーザーによるカスタマイズ | 見た目の変更は自由 | 見た目の変更は今後対応予定 |

**自由に並べ替えられるが浅い数字しか出せないダッシュボード**と、**並べ替えはまだできないが経営に直結する深い数字が出せるダッシュボード**。

事業成長に効くのは、間違いなく後者です。

stella-crmでは今後、「表示の自由度」も段階的に追加していきますが、**最大の強みである「データの深さ」を維持・拡張し続けることが最も重要**です。
