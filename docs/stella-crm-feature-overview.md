# Stella CRM — 機能概要書

> **本書の位置づけ**: 自社開発の統合業務管理システム「Stella CRM」の機能一覧と技術的特徴をまとめたものです。受託開発・技術パートナーシップのご提案にあたり、弊社の開発力・設計力をご理解いただくための資料です。

---

## 1. システム概要

Stella CRMは、**複数の事業ドメインを1つのプラットフォームで統合管理**するクラウド型業務システムです。

| 項目 | 内容 |
|------|------|
| **アーキテクチャ** | Next.js (App Router) + Prisma ORM + PostgreSQL |
| **インフラ** | Docker Compose によるコンテナ運用（VPS） |
| **データベース** | 150以上のテーブル、5,200行超のスキーマ定義 |
| **対応ドメイン数** | 4事業（人材派遣・保険代理店・補助金支援・経理） |
| **外部API連携** | 8サービス以上（CloudSign, MoneyForward, Google Slides, Telegram, LINE, ProLine 等） |
| **自動化CronJob** | 10種類以上の定期自動処理 |

## 2. 主要機能一覧

### 2.1 電子契約の完全自動化（CloudSign API連携）

契約書の作成から署名完了後の処理まで、**人手を介さず全自動で完結**します。

| 機能 | 詳細 |
|------|------|
| テンプレートからの契約書自動生成 | CloudSign APIでテンプレートを複製し、宛先・入力項目を自動セット |
| ワンクリック送信 | CRM画面から契約書を電子署名依頼として送信 |
| ステータス自動同期 | Webhook受信で「送信済→署名中→締結完了」をリアルタイム反映 |
| 締結済みPDF自動保存 | 署名完了後、PDF をサーバーに自動ダウンロード・保管 |
| 自社署名の自動追跡 | 自社側の署名完了も自動検知・記録 |
| リマインド機能 | 未署名の契約に対する自動リマインド送信 |
| 運営法人別テンプレート管理 | 複数法人でそれぞれ異なるテンプレートを設定可能 |

**技術ポイント**: Webhook + ポーリングの二重同期方式で、Webhook障害時も定期CronJobでステータスを補完。

---

### 2.2 請求書の自動生成・自動送信

請求書の作成から送信まで、契約データに基づいて**自動生成**されます。

| 機能 | 詳細 |
|------|------|
| 契約条件からの売上自動計算 | 初期費用・月額・成果報酬を契約データから自動算出 |
| インボイス制度完全対応 | 適格請求書番号、税率別集計（10%/8%/0%）、端数処理 |
| 請求書番号の自動採番 | `INV-YYYYMM-NNNN` 形式で法人別に安全に採番 |
| PDF自動生成 | 請求書データからPDFを自動生成 |
| メール自動送信 | テンプレートベースで宛先（TO/CC/BCC）を設定し一括送信 |
| 赤伝（クレジットノート）対応 | 修正請求書の自動発行 |
| 送信履歴管理 | 送信日時・再送・手動送信の全記録 |

### 2.3 受信請求書の自動取込・マッチング（IMAP連携）

取引先から届く請求書メールを**自動で取り込み、該当する支払いグループに紐づけ**ます。

| 機能 | 詳細 |
|------|------|
| メール自動監視 | IMAP経由で請求書メールを定期的にスキャン |
| PDF自動抽出 | メール添付のPDFを自動取得・保存 |
| 自動マッチング | ファイル名の参照コード（例: `PG-1234`）や送信元ドメインで自動判定 |
| 信頼度スコア | high / medium / low の3段階でマッチ精度を表示 |
| 未マッチアラート | 自動マッチできない場合は管理者にアラート |

## 3. 自動化フロー（代表例）

### フロー1: 契約締結の完全自動化

```
CRMで契約書作成
  → CloudSign APIでテンプレート複製・宛先設定
  → ワンクリックで電子署名依頼を送信
  → 相手方が署名
  → CloudSign Webhook → CRMステータス自動更新
  → 締結済みPDFを自動ダウンロード・保管
  → ProLine経由で関係者にLINE通知
```

### フロー2: 請求書発行〜入金確認

```
契約条件から売上レコードを自動生成
  → 請求書番号を自動採番
  → PDFを自動生成
  → メールテンプレートで一括送信
  → 銀行取引をMoneyForward APIで自動取込
  → 消込処理で入金を自動マッチング
  → 未回収は売掛金年齢表で可視化
```

### フロー3: 受信請求書の自動処理

```
取引先からメールで請求書が届く
  → IMAP CronJobで定期スキャン
  → PDF添付を自動抽出・保存
  → 参照コード/送信元ドメインで支払先を自動特定
  → 該当する支払グループに自動紐づけ
  → 管理者が最終確認して承認
```

### フロー4: リード獲得〜提案書送付

```
代理店ポータル経由でリードフォーム回答
  → CRMに自動取込・パイプラインに登録
  → ステージを「商談中」に更新
  → Google Slides APIで提案書を自動生成
  → シミュレーション結果を反映
  → PDFとして出力・送付
```
