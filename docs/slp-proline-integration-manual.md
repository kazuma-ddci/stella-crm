## 1. 全体像

### プロラインとは

**プロライン** は、公式LINEアカウントを使ったお客様対応を効率化するツールです。LINE上で「友達追加→アンケート→予約→自動メッセージ送信」までを行います。

### CRMとプロラインの役割分担

| ツール | 担当 |
|--------|------|
| **プロライン** | LINEのお客様対応（友達追加・予約フォーム・メッセージ自動送信） |
| **CRM（このシステム）** | 全データの集約と管理（友達情報・組合員管理・契約書・商談履歴） |

### 連携している主な処理

CRMとプロラインは、お客様がLINEで操作するたびに自動で情報をやり取りしています。

```
お客様の行動                    自動で起きること
────────────────────────────────────────────────────
LINE友達追加     →  CRMに「公式LINE友達情報」が自動登録
                     紹介者がいればその人にLINE通知

入会フォーム送信  →  CRMに「組合員」が自動登録
                     契約書（CloudSign）が自動送付

概要案内予約     →  CRMに「企業」と「商談予定」が自動登録
                     担当スタッフが自動で割り当て
                     紹介者にLINE通知

契約書に署名     →  組合員ステータスが「締結済み」に変更
                     LINEのリッチメニューが切り替わる
                     紹介者に「契約完了」通知が届く
```

## 2. 事前準備

連携を始める前に、以下4つの設定画面を一度だけセットアップします。

### 2.1 プロラインアカウント情報を登録する

**画面**: 設定 → **プロライン情報**（`/slp/settings/proline`）

公式LINEのアカウント情報を登録します。

| 入力項目 | 内容 |
|---------|------|
| 公式LINE名 | 例：「公的制度教育推進協会」 |
| ログインメール | プロライン管理画面のメールアドレス |
| ログインパスワード | プロライン管理画面のパスワード |
| ログインURL | プロラインのログインURL（短縮可） |
| 有効フラグ | 現在使っているアカウントだけ「ON」 |

複数のアカウントを登録できますが、**「有効フラグON」は1つだけ**にしてください。

### 2.2 プロライン担当者を CRMスタッフに紐付ける

**画面**: 設定 → **プロライン担当者**（`/slp/settings/proline-staff`）

プロラインの予約フォームには「概要案内担当者」という項目があります。お客様が予約したとき、CRMが「プロラインで田中太郎を選択した」という情報を受け取りますが、CRMはそれが **CRMのどのスタッフ** なのか分かりません。そこでこの紐付けが必要です。

| 入力項目 | 内容 |
|---------|------|
| プロライン担当者名 | プロラインに表示されている名前そのまま（例：「田中太郎」） |
| 紐付け先 LINE友達 | その担当者の公式LINE友達アカウント |
| 紐付け先スタッフ | CRMに登録されているスタッフ |

**注意**: プロライン側の名前と完全に一致する必要があります。「田中太郎」と「田中 太郎」（スペース有無）は別物として扱われます。

### 2.3 LINEに送るメッセージ文面を準備する

**画面**: 設定 → **通知テンプレート設定**（`/slp/settings/notification-templates`）

お客様や紹介者に自動送信するLINEメッセージの文面を、状況ごとに登録します。

代表的なテンプレートは以下の通りです。

| 文面の種類 | 送信される相手 | 送信されるタイミング |
|-----------|--------------|----------------|
| 概要案内お礼メッセージ | お客様 | 概要案内が完了したとき |
| 契約書未締結リマインド | 組合員本人 | 契約書を送付したのに未署名の場合 |
| 導入希望商談お礼メッセージ | お客様 | 導入希望商談が完了したとき |
| Zoom案内（概要案内） | お客様 | Zoom URLを送るとき |
| Zoom案内（導入希望商談） | お客様 | Zoom URLを送るとき |
| 概要案内 紹介者通知 | 紹介者 | 予約・変更・キャンセル・完了時 |

「初回」と「2回目以降」で文面を分けたり、「プロライン経由」と「手動送信」で分けたりもできます。

### 2.4 プロジェクト全体の動きを設定する

**画面**: 設定 → **プロジェクト設定**（`/slp/settings/project`）

| 設定項目 | 説明 |
|---------|------|
| 契約書自動送付 | 入会フォームが届いたら自動で CloudSign を送付するか |
| LINE後紐付け時の紹介者通知 | 契約締結後にLINE友達追加された場合、紹介者に自動通知するか |
| 入会フォーム用契約種別 | 自動送付する契約書のテンプレート |
| リマインド日数 | 契約書送付後、何日経っても未署名なら自動リマインドするか（例：7日） |

### 2.5 Zoom議事録のAI要約用プロンプトを登録する（任意）

**画面**: 設定 → **Zoom AIプロンプト設定**（`/slp/settings/zoom-ai-prompts`）

Zoom商談の議事録をAIで自動要約したり、AIにお礼メッセージを書かせたりする際の指示文を登録します。デフォルトのままでも動きますが、業務に合わせて調整できます。

## 3. 日々の運用

### 3.1 公式LINE友達情報を確認する

**画面**: **公式LINE友達情報**（`/slp/line-friends`）

公式LINEに登録された友達の一覧が見られます。3つのタブで構成されています。

#### タブ1: 公式LINE友達情報

プロラインから自動同期された友達データの全項目（LINE名、電話番号、住所、紹介者UIDなど）を確認できます。

- **プロライン由来のデータ**は読み取り専用（編集できません）
- 必要があれば「新規追加」ボタンで手動登録もできます
- **紹介者UIDが空欄**だと警告マークが出ます。紹介者がいない案件の場合は「紹介者不要」フラグでOFFにできます

#### タブ2: ユーザー情報

LINE友達と組合員ステータスを並べて確認できる、より見やすい画面です。

- 紹介者の名前が自動表示されます
- 組合員ステータス（「契約書送付待ち」「締結待ち」「組合員登録済み」など）が一目で分かります
- **代理店階層**（誰が誰を紹介したか）がツリー状に色分け表示されます

#### タブ3: AS管理

AS（アソシエイトスタッフ）を登録し、LINE友達アカウントとCRMスタッフに紐付けます。

### 3.2 概要案内予約の進捗を見る

**画面**: **企業名簿**（`/slp/companies`）

お客様が公式LINEから概要案内を予約すると、ここに自動で企業情報が登録されます。

- ステータスフィルタ：「予約中」「完了」「キャンセル」「飛び」で絞り込み可能
- 担当者は複数登録でき、主担当を指定できます
- 概要案内の日時・参加者・議事録・お礼メッセージ送信状況が記録されます
- ステータス変更時には理由を入力し、履歴が残ります

### 3.3 組合員（入会者）を管理する

**画面**: **組合員名簿**（`/slp/members`）

LINEの入会フォームから申し込みがあると、ここに自動登録されます。

| ステータス | 意味 |
|-----------|------|
| 契約書未送付 | フォーム受信済み、契約書はまだ送ってない |
| 契約書送付済 | CloudSignで契約書を送った |
| 組合員契約書締結 | お客様が署名完了 |
| 契約破棄 | 何らかの理由で破棄 |

**赤背景＋「✉️ 送信失敗」バッジ** が表示されている場合は、CloudSignのメール不達です。メールアドレスを修正して「新規契約書送付」ボタンで再送付してください。

主な操作ボタン：
- **新規契約書送付**: CloudSignで契約書を送る（個別 / 複数一括可）
- **リマインド送付**: 未締結者に CloudSign再通知 + LINE通知（テンプレート文）を送る
- **紹介者通知**: 紹介者に「○○さんが契約締結しました」とLINE通知（手動）

### 3.4 契約書を一覧で確認する

**画面**: **契約書管理**（`/slp/contracts`）

CloudSignで送付したすべての契約書のステータス（下書き / 送付済 / 締結 / キャンセル）と、送付日・締結日が一覧で確認できます。

### 3.5 リマインド対象を確認する

**画面**: **契約書リマインド**（`/slp/reminders`）

「契約書を送ったのに未署名」のお客様が一覧表示されます。一括でリマインド送付ができます。

毎日 朝10時に自動でもリマインドが実行されますが、手動で前倒しで送りたいときに使います。

## 4. 自動で起きていること

ここはスタッフが操作しなくても、システムが裏で自動的に行っている処理です。「なぜか自動でLINEが送られた」という現象の理由を理解するためのセクションです。

### 4.1 友達追加時の自動処理

お客様が公式LINEに友達追加すると：

1. プロラインからCRMに「友達追加情報」が届く
2. CRMの「公式LINE友達情報」に新規登録される
3. **紹介者UID（free1）が入っていれば**、紹介者のLINEに「○○さんが追加されました」と自動通知

### 4.2 入会フォーム送信時の自動処理

お客様がLINEの入会フォームを送信すると：

1. CRMの「組合員名簿」に新規登録される
2. プロジェクト設定で「契約書自動送付ON」なら、**CloudSignが自動送付される**
3. 「契約書管理」にも自動で記録される

### 4.3 概要案内予約時の自動処理

お客様がLINE上で概要案内を予約すると：

1. CRMの「企業名簿」に企業情報が自動登録される
2. 「商談予定」が登録される（日時・担当者付き）
3. 事前に設定した「プロライン担当者マッピング」を見て、**担当スタッフが自動で割り当てられる**
4. 紹介者に「○○さんが概要案内を予約しました」と自動通知

予約変更・キャンセルも同じ仕組みで自動処理されます。

### 4.4 契約書締結時の自動処理

お客様が CloudSign で署名すると：

1. CRMが CloudSignから通知を受け取る
2. 組合員名簿のステータスが「組合員契約書締結」に変わる
3. **公式LINEのリッチメニューが、組合員専用メニューに自動で切り替わる**
4. 設定で「ON」なら、紹介者に「○○さんが契約締結しました」と自動通知

### 4.5 毎日の自動リマインド（朝10時）

毎日朝10時に、以下が自動実行されます：

- 契約書送付から〇日経っても署名していないお客様を抽出
- CloudSignからリマインドメールを再送
- お客様のLINEにリマインドメッセージを送信

「リマインド除外フラグ」がONのお客様は対象外になります。

## 5. 困ったとき

### 5.1 まずここを見る ― 自動化エラーページ

**画面**: **自動化エラー**（`/slp/automation-errors`）

裏で自動実行されている処理がエラーで止まったとき、このページに表示されます。**毎朝チェックする習慣を持つことを推奨します。**

| タブ | 内容 |
|------|------|
| 未解決 | まだ対処していないエラー |
| 解決済み | 過去に対処したエラー |

各エラーには2つのボタンがあります：

- **もう一度実行**: もう一度自動処理を試します。原因が一時的な問題（ネットワーク不調など）なら成功します。成功すれば自動的に「解決済み」に移動します
- **解決済みにする**: 手動で「対処不要」とマークします

### 5.2 よくある問題と対処

#### ケース1: 組合員名簿に赤背景の人がいる

**原因**: CloudSignのメール送信が失敗しています。メールアドレスが間違っている可能性大。

**対処**:
1. 該当の組合員レコードを開く
2. メールアドレスを修正する
3. 「新規契約書送付」ボタンで再送付

#### ケース2: 紹介者にLINE通知が届かない

**原因の候補**:

1. **紹介者UIDが設定されていない**
   - 公式LINE友達情報の該当ユーザーで「free1」（紹介者UID）が空欄
   - 警告マークが出ているはず
2. **自動化エラーが発生中**
   - 自動化エラーページで「Form4」「Form5」「Form6」などのエラーを確認
   - 「もう一度実行」を押す

#### ケース3: 概要案内予約がCRMに反映されない

**原因の候補**:

1. **プロライン側のWebhook設定が外れている**（管理者に確認）
2. **自動化エラーが発生中**
   - 自動化エラーページで「概要案内予約Webhook」エラーを確認

#### ケース4: 担当スタッフが自動で割り当てされない

**原因**: プロライン担当者マッピングが未登録、または名前が不一致。

**対処**:
- 設定 → **プロライン担当者** の画面を開く
- プロライン側の担当者名と完全一致するか確認
- 不足していれば追加登録

#### ケース5: 契約締結後にLINE友達追加した人で通知が動かない

**原因の候補**:

1. リッチメニュー切替の自動処理が失敗
   - 自動化エラーページで「リッチメニュー切替」エラーを確認
2. 設定で「LINE後紐付け時の紹介者通知」がOFFになっている
   - 組合員名簿の該当者で「紹介者通知」ボタンから手動送信

## 6. 画面パス一覧

### 通常運用で使う画面

| 画面名 | パス | 主な用途 |
|--------|------|---------|
| 公式LINE友達情報 | `/slp/line-friends` | LINE友達の確認・代理店階層・AS管理 |
| 企業名簿 | `/slp/companies` | 概要案内予約企業の管理 |
| 組合員名簿 | `/slp/members` | 入会者管理・契約書送付・リマインド |
| 契約書管理 | `/slp/contracts` | CloudSign契約書一覧 |
| 契約書リマインド | `/slp/reminders` | リマインド対象一覧・一括送付 |
| 自動化エラー | `/slp/automation-errors` | エラーログ確認・リトライ |

### 設定画面（初期セットアップ用）

| 画面名 | パス | 設定する内容 |
|--------|------|-------------|
| プロライン情報 | `/slp/settings/proline` | プロラインアカウント情報 |
| プロライン担当者 | `/slp/settings/proline-staff` | プロライン担当者名⇔CRMスタッフの紐付け |
| 通知テンプレート設定 | `/slp/settings/notification-templates` | LINE通知の文面 |
| Zoom AIプロンプト設定 | `/slp/settings/zoom-ai-prompts` | AI議事録要約の指示文 |
| プロジェクト設定 | `/slp/settings/project` | 契約書自動送付ON/OFF等 |

## 付録: 朝のチェックリスト（推奨）

毎朝、以下を順にチェックすると運用上の問題を早期発見できます。

- [ ] **自動化エラー** ページに「未解決」エラーがないか
- [ ] **組合員名簿** に赤背景レコード（CloudSign送信失敗）がないか
- [ ] **契約書リマインド** ページで対象者が想定通りか
- [ ] **企業名簿** で「予約中」のうち、本日・明日の概要案内の準備ができているか
