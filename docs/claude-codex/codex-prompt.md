# Codex レビュープロンプト（コピペ用）

以下をCodexに貼り付けて使用する。毎回同じプロンプトでOK。
Codexは `claude-plan.md` を読んで評価し、`plan-review.md` を更新する。

---

## プロンプト本文（ここからコピー）

```
以下の2ファイルを読み込み、実装プランをレビューしてください。

- `docs/claude-codex/codex-plan.md`（元の要件・ロードマップ）
- `docs/claude-codex/claude-plan.md`（Claude Codeが作成した実装プラン）

### レビュー観点

1. **要件の網羅性**: codex-plan.md の要件が claude-plan.md に漏れなく反映されているか
2. **実装順序の妥当性**: Phase の優先順位・依存関係は適切か
3. **技術的な正確性**: テーブル設計、API設計、型定義に問題はないか
4. **リスク・懸念点**: 見落としているエッジケースや既存機能への影響はないか
5. **受け入れ条件の十分性**: 各Phaseの完了判定基準は明確か

### 出力先

レビュー結果を `docs/claude-codex/plan-review.md` に上書き保存してください。
前回のレビュー内容は残さず、最新の評価で上書きしてください。

### 重要な制約（レビュー時に考慮すること）

- `prisma db push` は禁止。スキーマ変更は `prisma migrate dev --name <変更内容>` を使う
- STPスコープのサーバー検証（UIだけに依存しない）は最優先で維持する
- 日本語文字列の `localeCompare` はクライアントコンポーネントで使用禁止（ハイドレーションエラー防止）
- VPSのDBに対するデータ消失操作は絶対禁止
```
